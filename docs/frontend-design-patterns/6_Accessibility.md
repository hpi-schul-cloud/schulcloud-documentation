# Accessibility (A11y)

The product should be usable by anyone — including people with disabilities, slow
connections, outdated hardware, or unfavorable environments.

## W3C Web Accessibility Initiative (WAI)

The WAI develops strategies, standards, and resources to make the web accessible.

- [Introduction to Web Accessibility](https://www.w3.org/WAI/fundamentals/accessibility-intro/)
- [ARIA Authoring Practices Guide (APG)](https://www.w3.org/WAI/ARIA/apg/)

WAI-ARIA extends web content and applications with attributes that identify
interactive features — enabling keyboard navigation and assistive technologies.

## Vuetify and Vue

Vuetify components provide accessible interactions for mouse-based actions and use
HTML5 semantic elements where applicable.

- [Vuetify A11y](https://vuetifyjs.com/en/features/accessibility/)
- [Vue A11y Best Practices](https://vuejs.org/guide/best-practices/accessibility.html)
